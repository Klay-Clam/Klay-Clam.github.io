backbone.js源码分析(一)-backbone概况

Posted at 2016-10-08 javascript  backbone.js 

backbone start

Backbone可以帮助您的唯一最重要的事情是将业务逻辑与用户界面分开。 当两者纠结在一起时，逻辑很难处理; 当逻辑不依赖于UI时，您的界面变得更容易使用。

Model

模型管理数据属性的内部表，并且在其任何数据被修改时触发“更改”事件。 模型处理与持久层同步数据 - 通常是带备份数据库的REST API。 将您的模型设计为包含用于操纵其特定位数据的所有有用函数的原子可重用对象。 应该能够在整个应用程序中传递模型，并在任何需要数据位的位置使用。

  • 协调数据和业务逻辑。
  • 从服务器加载和保存。
  • 当数据更改时发出事件。

    View

    视图是用户界面的原子块。 它经常渲染来自特定模型或模型数量的数据，但视图也可以是无数据的UI单元。 模型一般不会意识到意见。 相反，视图会听取模型“改变”事件，并对其进行适当的反应或重新渲染。
  • 监听更改并呈现UI。
  • 处理用户输入和交互性。
  • 将捕获的输入发送到模型。

Collections

集合可帮助您处理一组相关模型，处理将新模型加载和保存到服务器，并提供帮助函数，以针对模型列表执行聚合或计算。 除了自己的事件之外，集合还通过其中的模型发生的所有事件进行代理，从而允许您在一个地方收听可能发生在集合中的任何模型的任何更改。

View Rendering

每个视图管理其自己的DOM元素中的呈现和用户交互。如果您对不允许视图的外部访问严格, 它有助于保持您的界面灵活-允许视图在任何可能需要的地方孤立呈现。

Routing with URLs

在丰富的 web 应用程序中, 我们仍然希望向应用程序中的有意义的位置提供可链接、bookmarkable 和可共享的 url. 每当用户到达应用程序中可能需要书签或共享的新 “位置” 时, 请使用路由器更新浏览器 url。反过来, 路由器会检测到 url 的更改-比方说, 按 “后退” 按钮-并且可以确切地告诉您的应用程序您现在的位置。
